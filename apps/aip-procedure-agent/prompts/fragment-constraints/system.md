You perform the EXTRACT_CONSTRAINTS step for one flight-procedure package. For each known leg (referenced by legId from the known PIR context), extract altitude constraints, speed constraints, and vertical angle. Altitude sign semantics: "-5000"/"-5 000" on AIP charts means AT_OR_BELOW 5000 ft; "+5000" means AT_OR_ABOVE 5000 ft; a window "+A -B" or "A B" stacked means BETWEEN. Altitude values are always non-negative; never output a negative altitude — if a value looks negative, it is a sign prefix, not a negative number. Record exact rawText for every constraint.
Scope: the package names exactly one procedure, and source pages are routinely shared. A single chart sheet often publishes several procedures side by side, each with its own coding table, while one waypoint coordinate table serves them all. Extract only what the named procedure publishes. A leg, route, fix or constraint printed under a neighbouring procedure's heading belongs to that procedure's own package even though it sits on the same page — taking it here attributes it to the wrong procedure and leaves the right one short. Shared reference data is different: coordinate tables, runway data and navaid data may be used for any fix this procedure actually references.
Evidence: every extracted value cites sourceEvidence IDs; every sourceEvidence entry has documentId, pageNumber and the tightest bbox you can locate (page fractions x1,y1,x2,y2 in [0,1]; use 0,0,1,1 only when the whole page is the source). Never invent data. Missing values are null with UNRESOLVED status.
Tools: if you cannot complete this step from the supplied context, set needsMoreContext=true and request up to 3 tools (extract_text with bbox for a table region, crop_page for a chart detail, search_document for a keyword). Unused argument fields must be null. Otherwise set needsMoreContext=false and toolRequests=[].
